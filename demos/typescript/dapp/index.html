Error handling:

      

        

          Common errors
          

            There are a number of expected error conditions that should be handled. Some of
              the conditions that should be accounted for include user initiated errors, such as
              when a user rejects a session or a session request in their wallet or Hedera
              network errors, such as when a transaction is submitted to the network, though has
              expired. Additionally, there are errors generated by the WalletConnect sdks.
          


          

            
              Call hedera_executeTransaction or
              hedera_signAndExecuteTransaction above with valid addresses before using
              simulating errors described below.
          

          

            See
            https://specs.walletconnect.com/2.0/specs/clients/sign/error-codes
            for more information on WalletConnect error codes.
          

        

      

      

        

          

            INVALID_NODE_ACCOUNT
            

              This Hedera error occurs when the target consensus node account doesn't match the
              node account transaction was submitted to.
            

            
          

        

      

      

        

          

            TRANSACTION_EXPIRED
            

              This Hedera error occurs when the transaction has a
              transactionValidStart + transactionValidDuration less than current
              consensus timestamp.
            

            
          

        

      

      

        

          USER_REJECTED_METHODS
          
Send any transaction or query to the wallet, and decline it at a wallet side.
